# プロジェクト固有の行動指針 (GEMINI.md)

このファイルに記載された指示は、Gemini CLI のあらゆる動作に優先して適用されます。

## 1. コード修正の原則
- **独断での修正禁止**: 調査、分析、または質問に対する回答（Inquiry）において、ユーザーからの明示的な指示がない限り、ソースコードや設定ファイルを直接書き換えてはならない。
- **提案と実施の分離**: 修正が必要と判断される場合、まず調査結果と「修正案」を提示し、ユーザーの承諾（「実施してください」「修正して」等の具体的な Directive）を待つこと。
- **指示の厳守**: 修正方針についての指示があった場合は、指摘された内容のみを正確に反映し、関連しない余計な変更を加えないこと。

## 2. サーバー・インフラ操作
- **不必要な再起動の回避**: サーバーの再起動などは、環境変数の変更やビルドプロセスの都合上どうしても必要な場合にのみ、理由を添えて提案すること。
- **本番環境への配慮**: Stripe や Firebase などの本番環境設定に関しては、常に破壊的な変更（データ削除など）を避け、事前の確認を徹底すること。
