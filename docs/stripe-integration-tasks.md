# Stripe 本番移行に向けた修正事項リスト

このドキュメントは、Stripe 決済機能を本番環境へ移行する前に修正・改善が必要な項目を管理します。

## 1. セキュリティの強化
- [x] 1-1. `create-checkout-session` エンドポイントでの Firebase ID トークン検証の実装（`userId` の偽装防止）。
- [x] 1-2. サーバー側での二重サブスクリプション防止チェックの実装（既存購読者の再申し込みブロック）。

## 2. 解約・顧客管理ロジックの適正化
- [x] 2-1. サブスクリプション解除を「即時停止」から「期間終了時に解約（`cancel_at_period_end: true`）」に変更：
    - [x] 2-1-1. `/api/cancel-subscription` エンドポイントで `stripe.subscriptions.update` を使用し、即時キャンセルではなく「期間終了時に解約」を設定するように変更。
    - [x] 2-1-2. 解約実行時に `isPremium` を `false` にせず保持し、代わりに「解約予約フラグ（`cancelAtPeriodEnd: true`）」を Firestore に保存するよう修正。
    - [x] 2-1-3. Webhook（`customer.subscription.updated`）で `cancel_at_period_end` の変更を検知し、DB のフラグを同期する処理を追加。
    - [x] 2-1-4. 設定画面（`SettingsPanel`）で、解約予約中の場合に「終了予定日」と「期限まで利用可能」な旨を表示するよう UI を修正。
- [x] 2-2. アカウント削除および解約フローの修正：
    - [x] 2-2-1. `/api/cancel-subscription` エンドポイントから `stripe.customers.del`（顧客削除）を撤廃し、サブスクリプションのキャンセルのみを行う。
    - [x] 2-2-2. 退会（アカウント削除）時に、Firestore の `users/{userId}` ドキュメントを完全に削除する処理の実装/確認。
    - [x] 2-2-3. Stripe 側の顧客データは削除せず保持し、後日メールアドレスで特定・検索可能な状態を維持する。
- [x] 2-3. 有効期限内はプレミアム機能を維持し、期限が切れたタイミングで Webhook（`customer.subscription.deleted`）を通じて自動的にダウングレードする仕組みの徹底。

## 3. Webhook 処理の網羅性向上
- [x] 3-1. `customer.subscription.updated` イベントのハンドリング追加（サブスクリプション更新時の有効期限同期）。
- [ ] 3-2. `invoice.payment_failed` イベントのハンドリング追加（支払い失敗時の権限停止）。
- [x] 3-3. Webhook 処理内でのエラーハンドリングとログ出力の改善。

## 4. メール通知と外部サービス
- [ ] 4-1. Resend の送信ドメインを本番環境用（独自ドメイン）へ変更。
- [ ] 4-2. 決済完了メールの内容のブラッシュアップ。

## 5. フロントエンド UX の改善
- [ ] 5-1. 料金ページでのチェックアウトセッション作成中のローディングインジケーター表示。
- [x] 5-2. 設定画面での「解約予約済み（期間終了まで利用可能）」状態の適切な表示。
